# Сервис анализа объявлений

API на FastAPI: по описанию объявления возвращаются дополнительные микрокатегории (`detectedMcIds`), признак необходимости черновиков (`shouldSplit`) и тексты черновиков (`drafts`).

## Запуск приложения

Из корня репозитория, с установленными зависимостями:

```bash
pip install -r requirements.txt
uvicorn main:app --reload
```

После запуска документация Swagger: [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs). Эндпоинт анализа: `POST /analyze`.

Для работы GigaChat в `main.py` должны быть заданы корректные учётные данные.

## Проверка по API и метрики

С поднятым сервером (`uvicorn main:app --reload`) прогон по кейсам из `tests/test_responses.json`:

```bash
python tests/validate.py
```

Подробности формата данных и идей для тестирования — в [tests/README.md](tests/README.md).

Юнит-тесты расчёта метрик (без сервера и сети):

```bash
python tests/validate.py --unit
```

Исходная микрокатегория объявления (`mcId` из запроса) в метриках по **дополнительным** микрокатегориям **не участвует** (исключается и из эталона, и из ответа API).
